# wsdist-api

NestJS + MongoDB REST API that provides persistent storage for the `wsdist-web` FFXI damage simulator. Clients save and load individual equipment sets or full application states using short human-readable keys.

## Overview

All saved objects are identified by a **5-word key** (e.g. `brave-coral-swift-maple-stage`) generated by the server at save time. Keys are drawn from a ~512-word list of common English words (all ≤ 6 letters), giving ~3.5 × 10¹³ possible combinations — negligible collision probability in practice.

## Endpoints

All routes are prefixed with `/api`.

### Equipment Set

A single 16-slot gear loadout stored as a `slot → Name2` map.

| Method | Path | Description |
|---|---|---|
| `POST` | `/api/equipment-set` | Save a gear set; returns a new 5-word key |
| `GET` | `/api/equipment-set/:key` | Retrieve a gear set by key |

**POST `/api/equipment-set`**

Request body:
```json
{
  "slots": {
    "main":   "Naegling",
    "sub":    "Duban",
    "ranged": "None",
    "ammo":   "Chrono Bullet",
    "head":   "Adhemar Bonnet +1",
    "neck":   "Combatant's Torque",
    "ear1":   "Sherida Earring",
    "ear2":   "Telos Earring",
    "body":   "Adhemar Jacket +1",
    "hands":  "Adhemar Wristbands +1",
    "ring1":  "Petrov Ring",
    "ring2":  "Ilabrat Ring",
    "back":   "Camulus's Mantle",
    "waist":  "Ioskeha Belt +1",
    "legs":   "Samnuha Tights",
    "feet":   "Samnuha Coat"
  }
}
```

Response `201`:
```json
{ "key": "brave-coral-swift-maple-stage" }
```

**GET `/api/equipment-set/brave-coral-swift-maple-stage`**

Response `200`:
```json
{
  "slots": {
    "main": "Naegling",
    ...
  }
}
```

Response `404` if key not found.

### App State

Full simulator state including all four gear sets (tp1, ws1, tp2, ws2), job, weapon skill, enemy, abilities, and buffs. The `state` payload is an opaque JSON object — the API stores and returns it verbatim.

| Method | Path | Description |
|---|---|---|
| `POST` | `/api/app-state` | Save full app state; returns a new 5-word key |
| `GET` | `/api/app-state/:key` | Retrieve app state by key |

**POST `/api/app-state`**

Request body:
```json
{
  "state": {
    "mainJob": "cor",
    "subJob": "war",
    "masterLevel": 0,
    "wsName": "Savage Blade",
    "wsThreshold": 1000,
    "enemy": { "Name": "Apex Monster", "Level": 135, "Defense": 1000, ... },
    "abilities": { "Berserk": true },
    "tp1": { "main": "Naegling", ... },
    "ws1": { "main": "Naegling", ... },
    "tp2": { "main": "Naegling", ... },
    "ws2": { "main": "Naegling", ... },
    "buffs": { ... }
  }
}
```

Response `201`:
```json
{ "key": "tiger-verse-round-dense-plant" }
```

**GET `/api/app-state/tiger-verse-round-dense-plant`**

Response `200`:
```json
{ "state": { ... } }
```

Response `404` if key not found.

## Getting Started

### Prerequisites

- Node.js 20+
- A running MongoDB instance (local or Atlas)

### Setup

```bash
cp .env.example .env
# edit .env and set MONGODB_URI
npm install
```

### Run

```bash
# development (watch mode)
npm run start:dev

# production
npm run build
npm run start:prod
```

The server starts on `http://localhost:3000` by default (`PORT` in `.env`).

### Environment Variables

| Variable | Default | Description |
|---|---|---|
| `MONGODB_URI` | *(required)* | MongoDB connection string, e.g. `mongodb://localhost:27017/wsdist` |
| `PORT` | `3000` | HTTP port to listen on |

## Project Structure

```
src/
  app.module.ts              Root module — wires Mongoose and feature modules
  main.ts                    Bootstrap: global prefix, validation pipe, CORS
  common/
    wordlist.ts              ~512 common English words (≤ 6 letters each)
    key-generator.service.ts Generates random 5-word hyphen-joined keys
  equipment-set/
    equipment-set.schema.ts  Mongoose schema — { key, slots, createdAt }
    equipment-set.service.ts save() / get() with collision-safe key loop
    equipment-set.controller.ts POST / GET handlers
    equipment-set.module.ts
    dto/
      save-equipment-set.dto.ts  Validates incoming { slots } payload
  app-state/
    app-state.schema.ts      Mongoose schema — { key, state, createdAt }
    app-state.service.ts     save() / get() with collision-safe key loop
    app-state.controller.ts  POST / GET handlers
    app-state.module.ts
    dto/
      save-app-state.dto.ts  Validates incoming { state } payload
```

## Key Generation

Keys are generated by `KeyGeneratorService.generate()`:

1. Pick 5 words at random from `WORDS` (512 entries, all ≤ 6 letters).
2. Join with hyphens: `brave-coral-swift-maple-stage`.
3. Before persisting, check MongoDB for a collision; regenerate if found (astronomically rare).

The key space is 512⁵ ≈ 3.5 × 10¹³. At 1 million saves per day, exhausting the key space would take ~96,000 years.

## Notes

- `state` in app-state documents is stored as a raw BSON object — no schema validation beyond `IsObject()`. This keeps the API decoupled from the Vue app's internal state shape.
- Both collections have a unique index on `key` as a secondary safety net against concurrent collision races.
- CORS is enabled globally; restrict origins in production via `app.enableCors({ origin: '...' })` in `main.ts`.
